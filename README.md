社内向けレシピ管理アプリ（共通パスワード認証版）

- 全店舗のレシピをスマホで閲覧・編集・登録

- 写真付きで登録可能
- 全員でリアルタイム同期

## 🚀 セットアップ手順（約15分）

### ① Firebaseプロジェクト作成

1. [Firebase Console](https://console.firebase.google.com) にアクセス
2. 「プロジェクトを追加」→ 名前は何でもOK（例: `sanpo-recipe`）
3. Googleアナリティクスは「無効」でOK

### ② Firestore Database を有効化

1. 左メニュー「Firestore Database」→ 「データベースを作成」
2. **本番環境モード**を選択
3. ロケーションは `asia-northeast1`（東京）

### ③ Storage を有効化

1. 左メニュー「Storage」→ 「使ってみる」
2. **本番環境モード**を選択
3. ロケーションは `asia-northeast1`

### ④ ウェブアプリを登録して設定をコピー

1. プロジェクト設定（⚙️）→ 「マイアプリ」
2. ウェブアプリ追加 `</>`
3. 表示された `firebaseConfig` をコピー

### ⑤ index.html に貼り付け

`index.html` の `firebaseConfig` を自分の値に書き換え。

### ⑥ セキュリティルールを設定

`firestore.rules` と `storage.rules` の内容をFirebase Consoleの各「ルール」タブに貼り付けて公開してください。

> **⚠️ 注意**: ルールは「誰でも読み書き可」設定です。URLを社内のみで共有しアプリ側でパスワード認証するため、運用上問題ありません。

### ⑦ GitHubにアップ → GitHub Pagesで公開

既存の `staff-app` リポジトリに `recipe-app/` フォルダごとアップ。  
URL例: `https://19900220yuarakawa-ai.github.io/staff-app/recipe-app/index.html`

### ⑧ 完成！

URLにアクセス → パスワード `sanpo2026` でログイン → 「理解しました」 → 店舗を選んで開始

## 🔑 パスワードの変更方法（退職者対応）

1. ログイン画面下「⚙️ 管理者ログイン」をタップ
2. 管理者パスワード（初期: `admin2026`）を入力
3. 「ログインパスワード変更」で新しいPWを入力 → 変更
4. **残りのスタッフ全員に新パスワードを共有**（LINEなど）

退職した人は古いパスワードでログインできなくなります。

## ❓ FAQ

**Q. パスワードを忘れたら？**  
管理者パスワードでログインすれば確認・変更できます。両方忘れたらFirebase Consoleで `config/auth` を直接編集してください。

**Q. 写真がアップロードできない**  
Storageのルールが正しく設定されているか確認してください。

**Q. データのバックアップは？**  
Firebase Console の Firestore → エクスポート機能を月1回でも使えば安心です。

**Q. 50人を超えそう**  
100人くらいまでは無料枠で大丈夫。それ以上は Blazeプラン（従量課金、月数百円程度）への切り替えを検討。
